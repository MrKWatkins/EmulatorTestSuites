# M6502

[![NuGet Version](https://img.shields.io/nuget/v/MrKWatkins.EmulatorTestSuites.M6502)](https://www.nuget.org/packages/MrKWatkins.EmulatorTestSuites.M6502)
[![NuGet Downloads](https://img.shields.io/nuget/dt/MrKWatkins.EmulatorTestSuites.M6502)](https://www.nuget.org/packages/MrKWatkins.EmulatorTestSuites.M6502)

Test suites for MOS Technology 6502 emulators.

## Getting Started

Start by installing the [NuGet package](https://www.nuget.org/packages/MrKWatkins.EmulatorTestSuites.M6502) into your test project:

```
dotnet add package MrKWatkins.EmulatorTestSuites.M6502
```

You will then need to create a concrete implementation of the `M6502TestHarness` abstract base class. This is how you wrap up your emulator and expose it
through a common interface to the test suites.

## Harness Requirements

Your `M6502TestHarness` implementation needs to expose the machine state that the tests validate:

* registers `A`, `X`, `Y`, `S`, `P`, and `PC`;
* byte-level memory reads and writes;
* execution of a single instruction; and
* a test-framework specific failure mechanism via `AssertFail`.

The shared base class already provides:

* `TStates` for total cycles taken by the instruction;
* helpers such as `ReadWordFromMemory`, `WriteWordToMemory`, `CopyToMemory`, and `ClearMemory`;
* assertion batching through `CreateAssertionScope`; and
* cycle recording infrastructure through `RecordCycles`, `Cycles`, and `MutableCycles`.

When `RecordCycles` is enabled, your harness should append `Cycle` instances to `MutableCycles` in the order they occur and keep `TStates` in sync with the
instruction that just ran.

## Test Suite Types

The current 6502 package contains instruction-level suites. These execute one instruction at a time and compare the resulting processor state, memory, and
recorded bus activity.

Running the tests from a unit test framework should be straightforward; for example, using [NUnit](https://nunit.org/) with the
[Single Step](SingleStep.md) suite:

```c#
public sealed class SingleStepTests
{
    [TestCaseSource(nameof(TestCases))]
    public void SingleStep(SingleStepTestCase testCase) => testCase.Execute<MyM6502EmulatorTestHarness>();

    [Pure]
    public static IEnumerable<SingleStepTestCase> TestCases() => SingleStepTestSuite.Instance.GetTestCases();
}
```
